# What to Transfer to Another System

Short answer: **transfer nothing — just `git clone` (with Git LFS).** The trained
models, all outputs (figures, predictions, metrics) and the demo patches are now
**committed to the repo** (models via Git LFS), so a clone gives you the
**identical results** without retraining. See [`SETUP_WINDOWS.md`](SETUP_WINDOWS.md).

> ⚠️ **Install Git LFS first** (https://git-lfs.com) and run `git lfs install`
> before cloning — otherwise the `.pt` model files arrive as tiny pointer stubs.
> Fix afterwards with `git lfs pull`.

---

## What is in the repo vs. not

| Category | Location | In the repo? | Notes |
|----------|----------|-------------|-------|
| 🧠 Trained models | `outputs/checkpoints/*.pt` | ✅ Yes (Git LFS) | identical models after `git lfs pull` |
| 📊 Figures / predictions / curves | `outputs/figures/` | ✅ Yes | same images you see here |
| 📈 Metrics / reports / TensorBoard | `outputs/logs/`, `outputs/tensorboard/` | ✅ Yes | same numbers |
| 🧩 Demo patches + manifests | `data/processed/` | ✅ Yes | portable relative paths |
| 🐍 Virtual environment | `.venv/` | ❌ No | **rebuild** with `setup_env.ps1` (never copy) |
| 🗂️ Raw demo slides | `data/raw/` | ❌ No | regenerated by `--demo` (not needed) |
| 🔬 Real CATCH dataset | `data/raw/images/`, `data/raw/masks/` | ❌ No | download on the target machine (TCIA) |

## The normal workflow (transfer nothing)

On the other Windows system:

```powershell
git lfs install
git clone https://github.com/Dilawarkhaninfo/canine-melanoma-catch.git
cd canine-melanoma-catch
git lfs pull
powershell -ExecutionPolicy Bypass -File scripts\setup_env.ps1
.\.venv\Scripts\Activate.ps1
```

You now have the identical models, figures and metrics. To re-create outputs from
scratch instead, run the pipelines:

```powershell
python scripts\run_week1_2_pipeline.py --demo
python scripts\run_week3_4_pipeline.py --demo --quick
```

## When you *do* transfer files

### 1. To skip retraining (optional, saves ~1 hr on CPU)
Copy the two checkpoint files into the same folder on the other machine:

```
outputs\checkpoints\unet_baseline_best.pt
outputs\checkpoints\unet_resnet34_best.pt
```

Create the `outputs\checkpoints\` folder first if it does not exist. With these in
place you can evaluate or run predictions without training. (They are not required —
they regenerate if you train.)

### 2. The real CATCH dataset (the only genuine must-transfer)
This is the one thing that cannot be auto-generated. Two options:

- **Recommended:** download CATCH directly on the target machine from TCIA
  (see [`data/README.md`](data/README.md)). No transfer needed.
- **Or** copy your `data\raw\` folder across. Layout the pipeline expects:
  ```
  data\raw\images\   <- whole-slide images (.svs / .tiff / .png)
  data\raw\masks\    <- matching tumour masks (same filename per slide)
  ```
  Then run without `--demo`:
  ```powershell
  python scripts\run_week1_2_pipeline.py --input data\raw
  python scripts\prepare_segmentation_data.py --input data\raw
  python scripts\train_unet.py --arch unet
  ```

---

## Never transfer

- **`.venv\`** — it is tied to this machine's absolute paths. Always rebuild it on the
  new system with `scripts\setup_env.ps1`. Copying it will not work.

---

## How to send the optional files

For the trained models (~213 MB total), any of these works:
- a USB drive,
- cloud storage (Google Drive / OneDrive / Dropbox),
- `git`-based large-file hosting is **not** used here (files exceed GitHub's 100 MB limit).

Zip them to keep the folder structure:
```powershell
Compress-Archive -Path outputs\checkpoints\*.pt -DestinationPath checkpoints.zip
```
On the other machine, extract into `outputs\checkpoints\`.
