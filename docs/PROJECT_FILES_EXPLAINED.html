📁 Project Files Explained

Every file in simple words — what it is and what it does

Canine Melanoma · CATCH · DAIM2025A_088
🟢 = you RUN this 🔵 = helper code (runs by itself) ⚙️ = settings 📄 = document
How to read this: The project is organised into folders. config holds settings, src holds the actual code (split into small jobs), scripts holds the "run buttons", and docs holds documents. You normally only run the files in scripts.
🟢 The only files you actually RUN are these 2 (everything else is used by them automatically):
• scripts\run_week1_2_pipeline.py — prepares the images
• scripts\run_week3_4_pipeline.py — trains the AI model
Jump to: Root files config src/utils src/data_acquisition src/preprocessing src/data src/models src/training scripts docs data & outputs

📦 Root files (top folder)

The main project files you see first when you open the folder.
📄README.mdThe front page — what the project is and the quick commands to run it.
📄SETUP_WINDOWS.mdStep-by-step guide to install and run everything on a new Windows PC.
📄WHAT_TO_TRANSFER.mdTells you which files to copy to another computer (and which come from GitHub).
⚙️requirements.txtThe list of Python libraries the project needs (pip installs from this).
⚙️requirements-lock.txtThe exact library versions that are known to work (for safety/reproducibility).
⚙️environment.ymlSame library list, but for people who use Conda instead of pip.
⚙️.gitignoreTells Git which files to NOT upload (like the big virtual environment folder).
⚙️.gitattributesTells Git to store the big model files (.pt) using Git LFS (large file storage).

⚙️ config/

All the settings in one place — change here, not in the code.
⚙️config.yamlEvery setting: folder paths, quality limits, patch size (256), zoom levels (5/10/20), and training settings (learning rate, epochs, loss). Change a number here without touching the code.

🔧 src/utils/

Small helper tools used everywhere in the project.
🔵config.pyReads config.yaml and fixes all the folder paths so the code runs from anywhere.
🔵logger.pyPrints messages to the screen and saves them in a log file (so you can see what happened).
🔵seed.pyFixes the "randomness" so you get the same result every time you run (reproducible).
🔵demo_data.pyMakes fake (synthetic) microscope slides for Week 1–2, used when real data isn't there yet.
🔵demo_segmentation.pyMakes fake slides plus tumour masks for Week 3–4, so the U-Net has something to learn from.
🔵__init__.pyEmpty marker file that tells Python "this folder is a code package". (Every src folder has one.)

📥 src/data_acquisition/

Getting the real data and checking its quality.
🔵download_catch.pyExplains how to download the real CATCH dataset from TCIA, and checks the downloaded slides.
🔵quality_check.pyChecks each slide: is it blurry? too dark? enough tissue? Bad slides get rejected with a reason.

🧹 src/preprocessing/

The most important folder — cleaning and cutting the slides.
🔵image_io.pyReads and saves images — both normal pictures and huge whole-slide (.svs) files.
🔵tissue.pyFinds where the actual tissue is on a slide (ignores the empty white glass).
🔵stain_normalization.pyThe Macenko method — makes every slide the same colour, so the scanner's colour doesn't confuse the AI.
🔵patch_extraction.pyCuts the big slide into small 256×256 squares, at 3 zoom levels (5×/10×/20×), and pairs them with masks.
🔵dataset_split.pySplits the patches into train / validation / test groups (70 / 15 / 15).

🍱 src/data/

Feeding the prepared patches to the model.
🔵augmentation.pyRandomly flips, rotates and colour-changes training patches so the model sees more variety (learns better).
🔵segmentation_dataset.pyLoads each patch and its mask together and hands them to the model as a pair.

🧠 src/models/

The actual AI brain and how we score it.
🔵unet.pyThe U-Net model itself — two versions: a basic one (from zero) and a ResNet-34 one (pre-trained, transfer learning).
🔵losses.pyThe "score of wrongness" (BCE + Dice) — tells the model how far its answer is from the truth.
🔵metrics.pyThe report-card numbers: Dice, IoU, pixel-accuracy, Hausdorff — how good the model is.

🏋️ src/training/

Teaching the model and then testing it.
🔵trainer.pyThe training loop — the model learns round by round, the learning rate slowly drops, progress shows in TensorBoard, and it stops & saves the best model automatically.
🔵evaluate.pyTests the trained model on unseen patches and makes the result pictures (truth vs prediction) and graphs.

🟢 scripts/ (the "run buttons")

These join everything together — usually you only run files from here.
🟢setup_env.ps1One command to install everything (environment + PyTorch + libraries).
🟢run_week1_2_pipeline.pyWeek 1–2 in one go: quality check → colour fix → cut patches → split.
🟢prepare_segmentation_data.pyMakes the patches + masks dataset for Week 3–4 (with a safe slide-level split).
🟢train_unet.pyTrains and tests ONE U-Net, and saves figures + a report.
🟢run_week3_4_pipeline.pyWeek 3–4 in one go: prepare data → train baseline → train ResNet-34 → compare.
🟢make_deliverables.pyMakes the Week 1–2 Word report + PDF slides for the supervisor.
🟢make_deliverables_week3_4.pyMakes the Week 3–4 Word report + PDF slides.

📄 docs/

All the documents, reports and presentation guides.
📄Research_Proposal_*.html / .pdf / .docxThe original research proposal (the full plan of the whole project).
📄progress_report/The fortnightly progress reports (Week 1–2 and Week 3–4) — Markdown + Word.
📄slides/The supervisor presentation slide decks (PDF).
📄literature/Notes on the research papers behind the project.
📄video_guide_weeks1-4.htmlA visual walkthrough of Weeks 1–4 (English).
📄*_RomanUrdu.htmlRoman-Urdu video scripts: project walkthrough, what-happens-when-you-run, slide explanation.
📄PROJECT_FILES_EXPLAINED.htmlThis very page — the simple file dictionary you are reading now. 🙂

📂 data/ and outputs/ (created when you run)

These folders fill up automatically when you run the pipelines.
📂data/raw/The original slides (real CATCH slides, or fake demo ones).
📂data/processed/The cut patches, masks, and the train/val/test lists (manifests).
📂outputs/figures/The result pictures: stain before/after, sample patches, training graphs, predictions.
📂outputs/logs/The score reports and CSV/JSON summaries.
📂outputs/checkpoints/The trained model files (.pt) — the "saved brain".
📂outputs/tensorboard/The live training graphs data (open with the tensorboard command).
One-line summary 🎯 — config = settings · src = the brain & tools (small job-files) · scripts = the run buttons · docs = papers & guides · data & outputs = made when you run. To use the project, you only ever run the 2 pipeline files in scripts.
